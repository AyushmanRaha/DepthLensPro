# DepthLens Pro Engineering Audit — 2026-06-16

| Issue | Affected files/modules | Root cause | Implemented fix | Remaining limitation | Test / verification command |
|---|---|---|---|---|---|
| Native build matrix excluded Windows x64 and Linux x64 while scripts hard-coded ARM64 paths. | `package.json`, `electron-app/package.json`, `scripts/build-native-*.sh`, `scripts/build-native-windows.ps1`, `electron-app/scripts/verify-packaged-resources.js` | Target support was split across package scripts and shell scripts with ARM-specific defaults. | Added central platform target module, x64 build scripts, architecture arguments, ONNX build variants, and generalized packaged resource discovery text. | Cross-compiling still depends on electron-builder host/platform limitations. | `cd electron-app && npm test`; `pytest backend/tests/test_platform_targets.py` |
| macOS x64/universal needed explicit rejection after adding other x64 platforms. | `electron-app/src/platform-targets.js`, `electron-app/scripts/unsupported-arch.js` | Previous blocker rejected all x64 targets, including supported Windows/Linux x64. | Block only macOS x64/universal with clear messages; allow Windows/Linux x64. | None. | `node electron-app/scripts/unsupported-arch.js --platform darwin --arch x64` |
| Renderer settings were browser-local only and did not persist through Electron userData with corruption handling. | `electron-app/main.js`, `electron-app/preload.js`, `frontend/script.js` | The UI wrote broad settings directly to localStorage with no versioned app-level schema. | Added versioned sanitized settings, atomic write/rename, corrupt-file backup, narrow preload load/save bridge, and localStorage fallback. | Existing theme still uses its original localStorage key for early no-flash paint. | `node electron-app/test-persistence-schema.js` |
| Webcam depth could overwork thermally constrained laptops. | `frontend/script.js` | Fixed FPS scheduling did not account for high backend latency, and depth base64 was retained in hot webcam state. | Added adaptive FPS reduction/recovery, hidden-window pause behavior already enforced, lower active telemetry, and avoided retaining latest webcam base64. | Backend still returns base64 PNG for compatibility; binary streaming would be a larger API change. | Manual webcam smoke test; `rg "adaptiveFps" frontend/script.js` |
| RGB Camera detector errors were collapsed to “Detector unavailable.” | `backend/services/object_detection.py`, `backend/api/routes.py`, `frontend/script.js` | Optional TorchVision dependencies/weights/inference failures had one generic error payload and renderer message. | Added structured detector error codes/messages/actions and renderer-specific detector status rendering. | First detector weight download may still require network unless pre-cached by setup. | `pytest backend/tests/test_detector_errors.py` |
| ONNX standard builds failed too aggressively in resource expectations. | `electron-app/scripts/verify-resources.js`, `electron-app/scripts/verify-packaged-resources.js`, `package.json` | Standard and ONNX variants were not clearly separated in package scripts. | Standard verification remains optional; ONNX variants pass `--onnx require-all --models all` and fail before packaging. | Actual ONNX export is intentionally not run in CI to avoid large downloads. | `npm run verify:resources`; `npm run verify:onnx:required` when models exist |
| Benchmark/diagnostic workloads needed stronger bounds. | `backend/services/benchmarks.py` | User-supplied iterations could cause long runs. | Existing cap of 20 iterations retained and documented as thermal protection. | Heavier models can still be slow on CPU-only machines. | `pytest backend/tests/test_benchmarks.py` |
| Ground-truth metric cache needed to bypass image-only responses. | `backend/api/routes.py`, `frontend/script.js` | GT metrics are request-dependent and cannot reuse image-only cache entries. | Existing route bypass for `gt_file`/`gt_required` verified during audit; frontend passes `gt_file` to `/estimate` for GT mode and reads grouped metrics. | LPIPS remains unavailable when optional dependency support is absent. | `pytest backend/tests/test_ground_truth.py` |
| README platform/build documentation was ARM64-only. | `README.md` | Build docs predated Windows/Linux x64 workflow. | Added explicit platform support matrix and separate standard/ONNX 4-step clone/setup/build/launch blocks for macOS, Windows, and Linux. | Screenshots were not changed because UI layout was preserved. | `rg "Windows — standard build|Linux — ONNX build" README.md` |
